# 02.3 API Endpoints

> **Fuente de verdad:** El contrato completo de la API está definido en
> [`backend/src/main/resources/api/openapi.yaml`](../backend/src/main/resources/api/openapi.yaml).
> Los DTOs del paquete `com.thalassa.backend.dto` se **generan automáticamente** a partir
> de ese archivo mediante `openapi-generator-maven-plugin` al ejecutar `mvn clean compile`.
> **No edites los DTOs directamente; edita el YAML y regenera.**

> **🔒 Nota de Seguridad (JWT):** Todos los endpoints marcados como `AUTH` requieren la cabecera `Authorization: Bearer <token_jwt>`. Sin ella el servidor devuelve `401 Unauthorized`.

---

## ☕ Microservicio A: Java Spring Boot (Puerto 8080)

### Autenticación — `/api/auth`

| Método | Endpoint | Descripción | Auth | Código OK |
|---|---|---|---|---|
| `POST` | `/api/auth/register` | Crea la cuenta. Devuelve `UserResponse`. **No emite JWT.** | Público | `201` |
| `POST` | `/api/auth/login` | Autentica con email + password. Devuelve `AuthResponse` con JWT. | Público | `200` |

---

### Usuario — `/api/users`

| Método | Endpoint | Descripción | Auth | Código OK |
|---|---|---|---|---|
| `GET` | `/api/users/me` | Perfil del usuario autenticado. | AUTH | `200` |
| `PUT` | `/api/users/me` | Actualiza el precio del kWh (`electricityPriceKwh`). | AUTH | `200` |

**Body `PUT /api/users/me`:**
```json
{ "electricityPriceKwh": 0.18 }
```

---

### Acuarios — `/api/aquariums`

| Método | Endpoint | Descripción | Auth | Código OK |
|---|---|---|---|---|
| `GET` | `/api/aquariums` | Lista los acuarios del usuario (resumen). | AUTH | `200` |
| `POST` | `/api/aquariums` | Crea un acuario. Plan FREE: máximo 1. | AUTH | `201` |
| `GET` | `/api/aquariums/{id}` | Detalle completo: datos + equipo + fauna. | AUTH | `200` |
| `PUT` | `/api/aquariums/{id}` | Actualiza nombre, litros y tipo. | AUTH | `200` |
| `DELETE` | `/api/aquariums/{id}` | Elimina el acuario y todo su contenido (cascada). | AUTH | `204` |

**Body `POST/PUT /api/aquariums`:**
```json
{
  "name": "Arrecife Principal",
  "liters": 350,
  "type": "MARINO_ARRECIFE"
}
```
`type` acepta: `MARINO_PECES` | `MARINO_ARRECIFE`

### Fauna — sub-recurso de acuario y `/api/livestock`

| Método | Endpoint | Descripción | Auth | Código OK |
|---|---|---|---|---|
| `GET` | `/api/aquariums/{id}/livestock` | Lista la fauna del acuario. | AUTH | `200` |
| `POST` | `/api/aquariums/{id}/livestock` | Añade un espécimen. Devuelve `warning` si no es reef-safe en acuario arrecife. | AUTH | `201` |
| `PUT` | `/api/livestock/{id}` | Actualiza un espécimen. | AUTH | `200` |
| `DELETE` | `/api/livestock/{id}` | Elimina un espécimen (baja o venta). | AUTH | `204` |

**Body `POST/PUT livestock`:**
```json
{
  "name": "Pez Payaso Ocellaris",
  "category": "PEZ",
  "reefSafe": true,
  "quantity": 2,
  "speciesCatalogId": 7
}
```
`category` acepta: `PEZ` | `CORAL` | `INVERTEBRADO`  
`speciesCatalogId` es opcional (null para especímenes personalizados fuera del catálogo).

**Respuesta `POST /api/aquariums/{id}/livestock` (con warning):**
```json
{
  "livestock": { "id": 5, "name": "Pez Ballesta", "reefSafe": false, ... },
  "warning": "Advertencia: este espécimen no es reef-safe y el acuario es de tipo arrecife."
}
```
El campo `warning` sólo aparece cuando hay un problema de compatibilidad.

## Respuestas de error estructuradas

Todos los errores usan el formato `ErrorResponse`:
```json
{ "message": "Descripción del error.", "timestamp": "2026-04-14T10:00:00" }
```

| Situación | Código |
|---|---|
| Email o username ya en uso | `409 Conflict` |
| Credenciales incorrectas | `401 Unauthorized` |
| Recurso no encontrado o no pertenece al usuario | `404 Not Found` |
| Límite del plan freemium superado | `403 Forbidden` |
| Validación de campos fallida (`@Valid`) | `400 Bad Request` |
| Precondición no satisfecha (ej. kWh no configurado) | `422 Unprocessable Entity` |

---

## Flujo de autenticación

```
1. POST /api/auth/register  →  201  { id, username, email, subscriptionPlan }
2. POST /api/auth/login     →  200  { token, username, email, subscriptionPlan }
3. Todas las llamadas siguientes incluyen:
   Authorization: Bearer <token>
```
