## Sprint 1 — Cimientos (13–19 abr)

**Objetivo:** El backend arranca, la BD está creada y el sistema de autenticación funciona end-to-end.

### Tareas Independientes (pueden ejecutarse en paralelo)
- [ ] Crear proyecto Spring Boot con dependencias: `Spring Web`, `Spring Security`, `Spring Data JPA`, `MySQL Driver`, `Lombok`.
- [ ] Crear proyecto FastAPI Python con dependencias: `fastapi`, `uvicorn`, `requests`, `beautifulsoup4`, `fake-useragent`.
- [ ] Diseñar y crear el schema SQL en MySQL con todas las tablas definitivas.

### Tareas Dependientes (requieren las anteriores)
- [ ] Crear entidades JPA: `User`, `Aquarium`, `Livestock`, `Equipment`, `Wishlist`, `SpeciesCatalog`.
- [ ] Implementar `UserDetailsService` + filtro `JwtAuthFilter` en Spring Security.
- [ ] Implementar endpoints `POST /api/auth/register` y `POST /api/auth/login`.
- [ ] Poblar `data.sql`: 2 usuarios (FREE y REEFMASTER), ~20 especies en `species_catalog`.
- [ ] **Validación:** Login funciona en Postman y devuelve un token JWT válido.

## Sprint 2 — Lógica de Negocio Core (20–26 abr)

**Objetivo:** El CRUD completo de acuarios funciona, el límite freemium está activo y la alerta de compatibilidad está implementada.

### Tareas Independientes
- [ ] CRUD `AquariumController` + `AquariumService`: `GET`, `POST`, `PUT`, `DELETE /api/aquariums`.
- [ ] CRUD `LivestockController` + `LivestockService`: `GET`, `POST`, `DELETE /api/livestock`.
- [ ] CRUD `EquipmentController` + `EquipmentService`: `GET`, `POST`, `DELETE /api/equipment`.

### Tareas Dependientes
- [ ] Lógica freemium en `AquariumService.create()`: si `subscription_plan = 'FREE'` y ya existe 1 acuario → `403 Forbidden`.
- [ ] Lógica de compatibilidad en `LivestockService.add()`: si acuario es `MARINO_ARRECIFE` y `species_catalog.reef_safe = false` → respuesta con campo `warning`.
- [ ] Endpoint `GET /api/aquariums/{id}/energy` (solo REEFMASTER): calcula gasto mensual sumando `(watts/1000) × horas × 30 × precio_kwh`.
- [ ] Endpoint `PUT /api/users/me` para actualizar `electricity_price_kwh`.
- [ ] **Validación:** Postman collection completa con todos los casos de negocio (freemium bloqueado, alerta de reef, calculadora).

## Sprint 3 — Python + Integración + Frontend Base (27 abr – 3 may)

**Objetivo:** El scraper devuelve una lista de productos reales y el frontend tiene las vistas base conectadas al backend.

### Tareas Independientes
- [ ] Implementar lógica de scraping en Python: recibe `keyword`, devuelve lista JSON `[{name, price, img_url, product_url, store_name}]`.
- [ ] Setup React + Vite + TypeScript: estructura de carpetas, React Router, Axios, Tailwind/CSS.
- [ ] Definir DTOs TypeScript que espejeen los DTOs del backend.

### Tareas Dependientes (Python listo)
- [ ] Gestión de errores en el scraper: `TimeoutError` → lista vacía con `error_code`; `ParsingError` → ídem.
- [ ] Endpoint `GET /api/scraper/search?keyword=...` en Spring Boot que llama al Python con `WebClient` (timeout 8s).
- [ ] Variable de entorno `PYTHON_SERVICE_URL` en Spring Boot (no hardcodear `localhost`).

### Tareas Dependientes (React setup listo)
- [ ] Vistas: `LoginPage`, `RegisterPage` con gestión de token JWT en `localStorage`.
- [ ] Vista `DashboardPage`: grid de `AquariumCard` con botón "Crear acuario".
- [ ] Vista `AquariumDetailPage`: tabs para Fauna y Equipamiento.
- [ ] **Validación:** El flujo login → dashboard → detalle de acuario funciona en el navegador.
